🐳 Docker Setup (Recommended)

The Docker stack starts Django, PostgreSQL, Redis, Celery, and Elasticsearch with a single command.

# 1. Clone the repository
git clone https://github.com/drofji/django-snapadmin.git
cd django-snapadmin

# 2. Copy environment file
cp dist.env .env

# 3. Start the full stack
docker compose up --build

On first boot the app container will:

  • Run python manage.py migrate
  • Collect static files
  • Seed 50 demo objects via seed_demo
🎉 Ready! Admin UI: http://localhost:8000/admin/ (admin / admin)
Swagger: http://localhost:8000/api/docs/

💻 Local Setup

Uses SQLite by default — no external services required.

python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
cp dist.env .env   # Set DEBUG=True, leave POSTGRES_DB blank

python manage.py migrate
python manage.py seed_demo
python manage.py runserver
ℹ️ Graceful degradation Elasticsearch and Redis are optional for local development. All features degrade silently to DB-only mode when these services are unavailable.

🏗 SnapModel

Inherit from SnapModel to get automatic admin registration and a smart __str__.

from snapadmin import models as snap_models, fields as snap

class Product(snap_models.SnapModel):
    name = snap.SnapCharField(max_length=200, searchable=True)
    price = snap.SnapDecimalField(max_digits=10, decimal_places=2, filterable=True)
    available = snap.SnapBooleanField(default=True)

    class Meta:
        verbose_name = "Product"

🏷 Status Badges

Render coloured pill badges for any choice/status field directly in the admin list view.

from snapadmin import fields as snap

class Order(snap_models.SnapModel):
    status = snap.SnapCharField(
        max_length=20,
        choices=[("pending","Pending"),("shipped","Shipped"),("cancelled","Cancelled")],
    )

    status_display = snap.SnapStatusBadgeField(
        field_name="status",
        choices=[
            snap.SnapStatusBadgeFieldChoice(
                "pending",
                text_html_color="#856404",
                background_html_color="#FFF3CD",
                border_html_color="#FFEEBA",
            ),
            snap.SnapStatusBadgeFieldChoice(
                "shipped",
                text_html_color="#155724",
                background_html_color="#D4EDDA",
                border_html_color="#C3E6CB",
            ),
            snap.SnapStatusBadgeFieldChoice(
                "cancelled",
                text_html_color="#721C24",
                background_html_color="#F8D7DA",
                border_html_color="#F5C6CB",
            ),
        ],
    )

⚡ Function Fields

Computed columns — no database column, value derived from a callable at display time.

full_label = snap.SnapFunctionField(
    func=lambda obj: f"{obj.name} (${obj.price})",
    verbose_name="Label",
    show_in_list=True,
    safe_html=False,   # Set True to render HTML
)

🔑 API Authentication

All API requests require a token in the Authorization header:

Authorization: Token <40-character-token-key>

Token validation checks: token exists → is_active=True → not expired → user.is_active=True.

⚠️ Security Always serve the API over HTTPS. Token keys are equivalent to passwords.

🗝 Token Management

# Create a token
POST /api/tokens/
{
  "token_name": "CI Pipeline",
  "allowed_models": ["demo.Product", "demo.Order"],  // [] = all models
  "expires_in_days": 90                               // null = never expires
}

# Response (save the token_key now — not retrievable later!)
{
  "id": 1,
  "token_name": "CI Pipeline",
  "token_key": "Abc123...",   // 40 chars
  "expiration_date": "2025-05-27T00:00:00Z",
  "is_valid": true
}

Permissions

Each token combines two levels of access control:

  • Token scope — allowed_models list (empty = unrestricted)
  • Django permissions — standard view/add/change/delete per model

Both must pass for a request to succeed.

🌐 Model Endpoints

GET/api/models/schema/List all available model endpoints
GET/api/models/{app}/{Model}/List objects (paginated, 25/page)
POST/api/models/{app}/{Model}/Create a new object
GET/api/models/{app}/{Model}/{pk}/Retrieve a single object
PATCH/api/models/{app}/{Model}/{pk}/Partial update
DELETE/api/models/{app}/{Model}/{pk}/Delete object
# Example: list products
curl -H "Authorization: Token YOUR_TOKEN" \
     http://localhost:8000/api/models/demo/Product/

# Example: create a product
curl -X POST \
     -H "Authorization: Token YOUR_TOKEN" \
     -H "Content-Type: application/json" \
     -d '{"name": "New Item", "price": "29.99", "available": true}' \
     http://localhost:8000/api/models/demo/Product/

🌿 GraphQL API

SnapAdmin includes a dynamic GraphQL API powered by graphene-django. The schema is automatically built from your SnapModel subclasses.

GET/api/graphql/GraphiQL Playground (requires Token auth)
POST/api/graphql/GraphQL Query Endpoint

Dynamic Queries

For every registered model, two query fields are generated using the {app_label}_{model_name} naming convention:

TypeNaming ConventionExample
List Queryall_{app_label}_{model_name}all_demo_product
Detail Query{app_label}_{model_name}(id: Int!)demo_product(id: 1)
# Example: Fetching products and their customers via GraphQL
query {
  all_demo_order {
    id
    total
    customer {
      id
      firstName
      lastName
    }
  }
}
ℹ️ Authentication & Permissions GraphQL queries require the same Authorization: Token <key> header as the REST API. Permission checks are enforced at the resolver level.

🔍 Elasticsearch

Elasticsearch is entirely optional. When ELASTICSEARCH_ENABLED=False, every search operation silently falls back to an ORM icontains query.

from demo.search import search_products

# Works with or without Elasticsearch
results = search_products("laptop", limit=10)
# → [{id, name, price, available}, ...]

🌱 Demo Seeder

python manage.py seed_demo                   # 50 objects each
python manage.py seed_demo --count 200        # custom count
python manage.py seed_demo --flush            # wipe then re-seed
python manage.py seed_demo --no-index         # skip ES indexing

Set SNAPADMIN_AUTO_SEED=True in .env to seed automatically after the first migrate.
